 Ляля для тебя 
 1. Post creation:
 создание поста (название, текст, изобр., уникал. ID)
 если у поста нету комента она удаляется через 10 минут
 если комент есть она удаляется через 15 минут после последнего комента
 2. Коменты :
 можно коментить посты и чужие коменты
 комент должен показать (ID поста или комента на который он отвечает)   


Айка для тебя :
1. Session managment:
идинтификация через куки каждому юзеру присвоить уникальный аватар и имя полученные через Rick and Morty API
2. Avatar manag:
аватары уникальны но если закончатся то повторно
3. Images: 
все изоброжения храняться в s3 (triple-s) аватарки не сохраняем локально 


### Структура 

/internal
  /app
    /domain
      /models         // Модели данных, такие как Post, Comment и т.д.
      /ports          // Интерфейсы (порты), которые описывают операции (например, репозитории, сервисы)
      /services       // Реализация бизнес-логики, использующая порты
  /infrastructure
    /adapters
      /repository     // Реализация репозиториев, работающих с БД (например, PostgreSQL)
      /storage        // Адаптеры для работы с внешними сервисами (например, S3)
      /api            // Адаптеры для взаимодействия с внешними API (например, с Rick and Morty API)
  /interfaces
    /http
      /handlers       // HTTP-обработчики для обработки запросов от пользователей
      /middlewares    // Промежуточные слои (middleware), такие как аутентификация
    /grpc             // Если вы используете gRPC, здесь будет соответствующий код
